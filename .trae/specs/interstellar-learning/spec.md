# 星际学霸 - 产品需求文档

## Overview
- **Summary**: 一个面向青少年（13-18岁）的AI驱动学习生态系统，以"学习计划"为核心枢纽，通过模块间的有机联动，实现学习、习惯、心理健康的一体化管理，并参照Wiki方法论构建个人知识星球。
- **Purpose**: 帮助青少年建立完整的学习闭环——从计划制定、执行追踪到知识沉淀，形成可持续的学习成长飞轮。
- **Target Users**: 中学生（13-18岁）

## 设计理念：有机生态系统

本应用不是6个独立功能的堆砌，而是一个**自循环的学习生态系统**：

```
                    ┌──────────────┐
                    │   学习计划    │  ← 核心枢纽
                    │  (Planning)  │
                    └──────┬───────┘
                           │
       ┌───────────────────┼───────────────────┐
       │                   │                   │
       ▼                   ▼                   ▼
┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│  学习搭子    │◄─►│   番茄钟    │◄─►│  习惯养成    │
│  (Study Bot) │   │  (Pomodoro) │   │  (Habits)    │
└──────┬───────┘   └──────────────┘   └──────────────┘
       │                                      │
       │                                      │
       ▼                                      │
┌──────────────┐                              │
│  知识星球    │◄─────────────────────────────┘
│  (Wiki)     │  ← 知识沉淀与关联
└──────────────┘
       ▲
       │
┌──────────────┐
│  心情日记    │  ← 情绪影响学习效率
└──────────────┘
```

## 模块依赖关系

### 核心枢纽：学习计划
- **依赖**：心情日记（了解用户状态）→ 调整计划难度
- **被依赖**：
  - 番茄钟（根据计划分配任务）
  - 习惯养成（计划任务转化为习惯）
  - 学习搭子（计划中遇到问题时求解）

### 学习搭子
- **依赖**：
  - 知识星球（基于已有知识回答更精准）
  - 学习计划（了解当前学习目标）
- **被依赖**：番茄钟（学习中随时调用）

### 知识星球
- **依赖**：学习搭子（一键保存对话中的知识点）
- **被依赖**：学习搭子（回答时参考知识库）
- **特点**：采用Wiki风格，支持知识点间的链接和关联

### 番茄钟
- **依赖**：学习计划（从计划中获取当前任务）
- **被依赖**：
  - 习惯养成（番茄完成计入习惯打卡）
  - 学习搭子（学习时随时提问）

### 习惯养成
- **依赖**：番茄钟（执行习惯任务）
- **被依赖**：学习计划（计划任务转化为习惯）

### 心情日记
- **依赖**：无（独立模块）
- **被依赖**：学习计划（情绪影响计划制定）

## Goals
1. **核心枢纽**：建立以"学习计划"为中心的任务分发体系
2. **有机联动**：实现6大模块间的数据流通和功能协作
3. **知识沉淀**：参照Wiki方法论构建个人知识星球
4. **情感陪伴**：AI学习搭子提供温暖的学习陪伴
5. **习惯养成**：通过番茄钟和习惯追踪培养自律

## Non-Goals (Out of Scope)
- 不提供K12课程内容（用户自主学习）
- 不涉及在线支付功能
- 不开发移动端原生应用（H5跨平台）
- 不包含社交分享功能（隐私优先）

## Background & Context
- 项目基于TRAE AI创造力大赛参赛要求设计
- 接入火山引擎ARK Coding Plan（minimax-m3模型）
- 采用星空主题H5设计，适配手机浏览器
- 参照Wiki维护手册构建知识沉淀体系

## Functional Requirements

### FR-1: 学习计划（核心枢纽）
- **核心功能**：
  - AI根据学习目标、可用时间、科目生成个性化学习计划
  - 计划自动拆解为每日任务
  - 根据心情状态调整任务难度（心情好→挑战题，心情差→轻松任务）
- **输出**：
  - 每日学习任务列表（推送给番茄钟和习惯养成）
  - 周期复盘（周末总结本周完成情况）
- **数据流**：
  - 输入：心情数据、学习目标、时间安排
  - 输出：任务列表 → 番茄钟、习惯养成

### FR-2: AI学习搭子
- **核心功能**：
  - 多科目学习对话，流式输出
  - 不直接给答案，引导思考
  - **一键保存**：对话中重要的知识点自动存入知识星球
  - **上下文感知**：了解当前学习计划和任务
- **数据流**：
  - 输入：知识星球数据、学习计划当前任务
  - 输出：对话记录、知识卡片（可选保存）

### FR-3: 番茄钟
- **核心功能**：
  - 从学习计划获取当前任务
  - 多种时长模式（15/25/45分钟）
  - 环形进度条可视化
  - **边学边问**：番茄钟期间可随时调用学习搭子
  - 完成计入习惯打卡
- **数据流**：
  - 输入：学习计划当前任务、习惯养成待打卡项目
  - 输出：完成记录 → 习惯养成、学习计划统计

### FR-4: 习惯养成
- **核心功能**：
  - 习惯创建（与学习计划任务关联）
  - 每日打卡（结合番茄钟完成）
  - 连续天数统计与激励
  - 根据学习计划推荐习惯
- **数据流**：
  - 输入：学习计划任务
  - 输出：打卡记录 → 学习计划完成统计

### FR-5: 心情日记
- **核心功能**：
  - 每日心情打卡（5种情绪状态）
  - AI情绪分析与温暖回应
  - 7天心情轨迹可视化
  - **影响学习计划**：心情数据推送给学习计划生成器
- **数据流**：
  - 输出：情绪数据 → 学习计划（调整任务难度）

### FR-6: 知识星球（Wiki风格）
- **核心功能**：
  - **多维度分类**：按科目、知识点、类型（概念/方法/题目）分类
  - **知识点关联**：支持知识点间的`[[链接]]`，构建知识网络
  - **AI学习总结**：自动提炼关键知识点
  - **一键收藏**：从学习搭子对话中保存
  - **学习搭子参考**：回答时自动检索相关知识
- **数据流**：
  - 输入：学习搭子对话笔记、AI总结
  - 输出：知识网络 → 学习搭子回答参考

## 模块间数据流详解

```
用户打开应用
    │
    ├─► 心情日记 → 今日心情 → 推送给【学习计划】
    │                              │
    │                              ▼
    │                    根据心情调整任务难度
    │                              │
    │                              ▼
    │                    生成【今日学习任务】
    │                              │
    │         ┌───────────────────┼───────────────────┐
    │         ▼                   ▼                   ▼
    │   【番茄钟】          【习惯养成】         【学习搭子】
    │         │                   │                   │
    │         │从计划获取任务      │从计划转化习惯      │了解当前任务
    │         │                   │                   │
    │         │完成计时           │打卡成功            │对话学习
    │         │                   │                   │
    │         │边学边问 ──────────┼───────────────────┘
    │         │                   │
    │         │番茄完成 ──────────┼──► 计入习惯打卡
    │         │                   │
    │         └───────────────────┴──► 【学习计划】完成统计
    │                                           │
    │                                           ▼
    │                               ┌────────────────────┐
    │                               │    知识星球沉淀     │
    │                               │  （重要知识点保存）   │
    │                               └────────────────────┘
```

## Non-Functional Requirements
- **NFR-1**: 响应时间 < 200ms（非AI请求）
- **NFR-2**: AI流式响应延迟 < 1s
- **NFR-3**: **H5移动端优先**：适配320px-428px宽度屏幕
- **NFR-4**: 触摸友好的交互设计（大按钮、手势支持）
- **NFR-5**: 数据本地存储（localStorage），支持数据导出
- **NFR-6**: 安全合规（不收集敏感信息）

## Constraints
- **Technical**: Node.js 18+, Express, 火山引擎ARK API
- **Design**: H5移动端优先，星空主题
- **Dependencies**: 火山引擎ARK API Key

## Assumptions
- 用户主要使用手机浏览器访问
- 用户具备基本的触屏操作能力
- 用户网络环境稳定

## Acceptance Criteria

### AC-1: 模块联动 - 学习计划驱动番茄钟
- **Given**: 用户已制定今日学习计划
- **When**: 打开番茄钟
- **Then**: 自动显示计划中的第一个任务，用户可选择开始或跳过
- **Verification**: `programmatic`

### AC-2: 模块联动 - 学习搭子上下文感知
- **Given**: 用户正在执行学习计划中的数学任务
- **When**: 调用学习搭子提问
- **Then**: AI了解当前任务背景，给出更精准的指导
- **Verification**: `programmatic`

### AC-3: 模块联动 - 番茄完成计入习惯
- **Given**: 用户完成一个番茄钟
- **When**: 番茄钟结束
- **Then**: 如果该任务关联了习惯，自动打卡成功
- **Verification**: `programmatic`

### AC-4: 模块联动 - 一键保存知识
- **Given**: 用户与学习搭子对话
- **When**: 点击"保存到知识星球"
- **Then**: 知识点存入知识星球，并建立学科关联
- **Verification**: `programmatic`

### AC-5: 模块联动 - 心情影响计划
- **Given**: 用户心情为"terrible"
- **When**: 生成学习计划
- **Then**: 计划中减少高难度任务，增加轻松任务
- **Verification**: `programmatic`

### AC-6: 知识星球 - Wiki风格链接
- **Given**: 用户查看知识点"勾股定理"
- **When**: 点击关联知识点"直角三角形"
- **Then**: 跳转到对应知识点页面
- **Verification**: `programmatic`

### AC-7: H5移动端适配
- **Given**: 在手机浏览器打开
- **When**: 正常操作
- **Then**: 界面适配良好，触摸操作流畅
- **Verification**: `human-judgment`

### AC-8: 整体体验
- **Given**: 用户完成一天的学习
- **When**: 从心情→计划→番茄钟→习惯→知识沉淀的完整流程
- **Then**: 各模块数据流通顺畅，形成学习闭环
- **Verification**: `human-judgment`
